# R3 — Surveiller les CT logs

**Explication** : Les Certificate Transparency (CT) logs sont des journaux publics qui enregistrent tous les certificats TLS émis. Leur surveillance permet de détecter l'émission frauduleuse de certificats pour un domaine. L'exploitant doit vérifier régulièrement qu'aucun certificat illégitime n'a été émis pour ses domaines.

**Statut** : ❌ Non conforme
**Vérification** : Effectuée le 2026-02-09

**Constat** : Aucun mécanisme de surveillance des CT logs n'est en place. Pas d'en-tête `Expect-CT` configuré. Aucun outil de monitoring (certspotter, crt.sh) ni script de vérification de certificats. Aucune alerte CloudWatch/SNS pour les événements ACM. Les pipelines CI/CD (`.gitlab-ci.yml` + 7 fichiers dans `.gitlab/`) ne contiennent aucune tâche de surveillance de certificats. Les certificats ACM sont gérés par AWS mais sans monitoring.

**Preuves/Fichiers** :
- `infrastructure/preliq/kustomize/overlays/root/ingress.yaml` — ARN ACM sans monitoring
- `.gitlab-ci.yml` + `.gitlab/*.gitlab-ci.yml` — aucune tâche CT logs
- `docker/nginx/nginx.conf` — aucun en-tête Expect-CT
- Domaines à surveiller : `preliq.dev.cegape.fr`, `e2e-preliq.dev.cegape.fr`
