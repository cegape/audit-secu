# R1 — Mettre en oeuvre TLS à l'état de l'art

**Explication** : Le site doit utiliser le protocole TLS (HTTPS) avec des suites cryptographiques conformes à l'état de l'art (TLS 1.2 minimum, TLS 1.3 recommandé). Les algorithmes obsolètes (RC4, 3DES, SHA-1) doivent être désactivés. Les certificats doivent être émis par une autorité de confiance et avoir une durée de validité raisonnable.

**Statut** : ⚠️ Partiel
**Vérification** : Effectuée le 2026-02-09

**Constat** : TLS activé en production via AWS ALB (port 443) avec certificats ACM. Cependant : aucune politique SSL explicite (`ssl-policy`) dans les annotations ingress (défaut AWS potentiellement obsolète), aucune redirection HTTP→HTTPS forcée, configuration HTTPS commentée dans application.yml (lignes 112-117), environnement Docker/dev en HTTP uniquement (nginx sur port 80, Tomcat sur 8080 sans connecteur HTTPS).
