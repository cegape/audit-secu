# R5 — Dissocier clairement la composition des pages web

**Explication** : Le code HTML, CSS et JavaScript doit être séparé dans des fichiers distincts. Le JavaScript inline (dans les attributs `onclick`, `onload`, etc.) et le CSS inline doivent être proscrits. Cette séparation facilite la mise en place de politiques CSP strictes et réduit la surface d'attaque XSS.

**Statut** : ⚠️ Partiel
**Vérification** : Effectuée le 2026-02-09

**Constat** : Sur 747 fichiers GSP analysés : **0 bloc `<script>` inline** et **0 bloc `<style>` inline** (excellent). Cependant : **27 handlers JS inline** (onclick, onchange) dans 19 fichiers (2.5%), concentrés sur le requêteur avancé et les modales de création dossier paie. **506 attributs CSS inline** (`style=`) dans 260 fichiers (34.8%), principalement pour le dimensionnement de colonnes de tableaux (`width: X%`) et alignements (`text-align: center`).
